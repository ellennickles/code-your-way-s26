## Assignment

### Creative Exercises Path • Refactor with Git Branches

<details>

<summary>Assignment details</summary>

The ideas this week are to practice refactoring code you've already written and to use Git branches as a safe way to experiment with changes.

**Tip:** Consider installing the [Oh My Zsh shell framework](https://github.com/ellennickles/code-your-way-s26/blob/main/version-control-guides/tips-and-tricks.md#oh-my-zsh) (if not already implemented) to automatically display when you are in a Git repository along with the current branch.

#### Part 1: Refactor + Git Branches

- **Set up your local project**
  - Create a local directory for a p5.js project and initialize it as a [Git
    repository](https://github.com/ellennickles/code-your-way-s26/blob/main/version-control-guides/git.md#create-a-git-repository).
  - Choose a previous sketch (from this course or another) to refactor so that
    the output remains the same, but the code is restructured for clarity and
    efficiency.
  - If your sketch is in the p5 web editor, copy and paste the code into your
    local p5 project.
- **Refactor your sketch**
  - Challenge yourself to refactor a sketch you really enjoyed but found messy,
    whether due to time constraints, evolving understanding of programming
    concepts, or a hunch that it could be optimized (e.g., addressing "code
    smells").
  - As you refactor, commit notable changes or milestones. Consider: reducing
    redundancy (loops, functions, or classes), cleaning up unused code,
    improving readability (rename variables and functions for clarity), adding
    comments, and organizing files (see Coding Train video above).
- **Use Git branches**
  - Create at least one new branch to refactor part of your project.
  - Practice switching between your branch and `main`.
  - Merge your branch into `main` when you're ready.
- **Bonus: Push to GitHub**
  - Practice what we covered in the second half of class, and create a remote
    repository on GitHub and push your refactored project. See our [GitHub
    guide](https://github.com/ellennickles/code-your-way-s26/blob/main/version-control-guides/github.md)
    for the steps.
  - If you get it working, share the link in your documentation instead of the p5 web editor link!

#### Part 2: Document

- Link to your original AND refactored code.
- Describe the changes you made and why you made them.
- How did it go with Git branches? Did branching help you experiment more
  freely? If not, what might be a use case for branching in your future work?
- Where did you get stuck, whether with Git, coding, or both? What steps did you
  take to troubleshoot?

### Self-Guided Study Path • Reflect and Reorient

<details>

<summary>Assignment details</summary>

This week's focus is on taking stock of where you are and where you're headed.

#### Part 1: Keep Building

Continue working through your resources and building on last week's progress. Aim to complete the next section or module and apply what you're learning through a small exercise, experiment, or mini-project.

If you're finding that your resources aren't quite right (too basic, too
advanced, or not what you expected) that's useful information. Make a note of it
and reach out to me so we can find alternatives together.

#### Part 2: Document

This is the most important part of your assignment this week. In Week 8, everyone will share project proposals with the class for feedback. Use this week's documentation to take an honest look at where things stand and what you're building toward.

- **What did you work on this week?** Topics, exercises, mini-projects, etc.
- **What clicked?** Concepts or techniques that made sense or felt useful.
- **What was challenging?** Points of confusion, how you approached them, and
  whether you found a solution or decided to revisit later.
- **What's your trajectory?** Based on what you've learned so far, what are you
  thinking for your project? Is the direction from your original plan still
  holding, or is it shifting? Either is fine, what fits best?
- **What do you need?** Any resources, tools, or support that would help you
  move forward?
